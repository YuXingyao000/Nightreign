# From 战斗策划心智模型

这是关于"他们脑子里是怎么想 boss 的"的笔记，不是关于具体代码的。

---

## 核心句

> **这套架构不是在描述 boss "怎么想"，而是在描述 boss "在每个边界面对玩家时该呈现什么样的概率分布"。**

行为树是确定性的（同输入同输出）；这套是**有节制的不确定性**。"有节制"是关键 —— 不是真随机，而是**根据情境调整的概率分布**。

---

## 设计 boss 的顺序（猜测的）

### 1. 先列出玩家在每个距离段会做什么

```
0-5m   : 贴脸输出 / 绕背
5-10m  : 试探性靠近 / 翻滚到位
10-20m : 调整位置 / 喝药 / 等机会
20+m   : 拉距离 / 远程攻击 / 等读条
```

### 2. 为每个距离段设计"对应玩家行为的招式池"

```
0-5m  : 爆发 AOE（应对贴脸）+ 转身斩（应对绕背）
5-10m : 突进招（追玩家）+ 短距斩（教训试探）
10-20m: 标准近战追击 + 中距投射
20+m  : 大范围远程 + 突进位移
```

**这一步在代码里 = `Activate` 的距离段权重池。**
**每段权重池反映策划想让玩家在那个距离感受到什么压力。**

### 3. 为"读招/被读招"博弈设计概率门限

玩家学会某招会怎么破解？**翻滚一次**。
- 70% 接 combo → 让"翻滚一次后立刻反击"成为陷阱
- 30% 不接 combo → 让"老老实实翻滚 + 反击"成立

**70/30 不是随便定的，是"让新手受罚 / 让老手有路可走"的平衡点。**

### 4. 为每个攻击设计呼吸窗口

每招打完都要留反击空间 — 魂系公约。
**`AfterAttackAct` 写进框架本身**，不是逐个 boss 调 — 这是团队规则。

### 5. 用 SE 标签编码"刚才发生了什么"

策划想让 boss "记得"刚才做过什么，但 lua 没真正的状态变量。
解决：用 boss 自己身上的 SE 当状态位。
- 刚出过远程 → 上身 60555
- 进入 combo 第 1 段 → 上身 60551
- 蓄力被打断 → 上身 60592

**副作用**：AI 状态 = 玩家可观察状态。SE 同时驱动光效/属性变化。**统一让 debug 和 modding 都容易。**

### 6. 决定哪些招可以"中途变招"

**只有"打完后玩家有反击机会"的招才接 combo** — 接 combo 是为了消除那个反击机会。
- 远程蓄力（玩家躲过会冲上来）→ 接近战收尾
- 普通近战 1 → 70% 接近战 2

**Interrupt 的设计原则**：消除特定的玩家应对策略。
- 玩家会翻滚？在翻滚刚结束的距离上设接续招
- 玩家会绕背？在动画后段加广角扫描招（如 360° 检测）
- 玩家会拉开距离？接远程突进

## 五条隐含设计契约

读完代码我总结出 boss 体感来自这五条约定：

### 契约 1：决策边界 = 目标切换边界
lua 只看一个 `TARGET_ENE_0`。引擎只在 Activate 入口换目标。
→ 多人时 boss 打完一段连段才转头看新玩家，而非中途甩头
→ "转头"成为语义化的预告信号，玩家能学会读

### 契约 2：博弈感来自决策密度，不是真智能
boss 不读你的输入，只读你的位置 + 投骰子。
但每招有 6-10 个决策点（动画关键帧）。
→ 玩家感受是"它在反应"
→ 熟练 = 学会读这些视觉关键帧

### 契约 3：呼吸窗口是独立 Goal
没把"打完冷静 N 秒"写成时间常量，而是独立 Goal + 空 Activate/Update。
→ 调难度只要改这个 Goal 是否被推入
→ 优雅解耦

### 契约 4：变招 = 槽位不变，AttackID 在 Act 里 swap
Act16/17 等函数根据 SE 替换 AttackID（3026→3028→3030）。
→ 二阶段"视觉差不多但有新动画"的来源
→ 权重池结构跨阶段稳定，便于调参

### 契约 5：朝向锥是软切池
`IsInsideTargetCustom(前方120°)` 失败时整个攻击池清零，只剩走位 slot。
→ 绕背玩家让 boss 卡在转向+侧步循环
→ 这是 lua 层对"绕背稳定策略"的回应（不是阻止）

## 玩家视角问题 ↔ 设计回应

| 玩家感受 | 代码对应 |
|---|---|
| "她虽然疯狂但能打" | `AfterAttackAct` + 走位 slot（14-37% 权重）|
| "为什么熟练后能打？" | 视觉锚点（帧标签 + 呼吸）+ 30% Replan 是熵的来源 |
| "她在凝视我" | Interrupt 在每个帧标签上重读玩家位置 |
| "为什么有时翻滚被秒有时能反击？" | 70/30 概率门限 |
| "为什么多人不混乱？" | 契约 1（目标切换只在决策边界）|

## 工具的选择

> "其实不管 HFSM 还是 BT，都是为了达到战斗策划心里想要玩家获得的体验。
>  只有最'适合'的工具，不一定一比一复刻代码。"

这是对的。From 用 Goal Stack + Weighted Selector 是因为它适合表达"承诺感 + 概率分布"。
BT 也可以表达类似效果，但要绕一些弯路（需要装饰器+冷却+权重节点）。
StateTree 也能做。

**重要的不是工具，是先问自己：**
- 这只 boss 该有几个"宏观状态"？
- 在战斗中，按什么维度切池子？（距离 / HP / 上一招）
- 每个池子里的权重反映了什么压力？
- 哪些招应该接 combo？接 combo 是为了消除什么玩家策略？
- 每招后给玩家多长反击窗口？

把这些问题想清楚再决定工具。**工具应该服从设计意图，而不是反过来。**

---

## 一个有点反直觉的结论

**复刻 boss 时，不要试图让它更聪明。试图让它更克制、更稳定、更可被学习。**

From 的天才在于克制：
- boss 不读你的输入
- 永远只看你的位置
- 永远在 70/30 这种概率上做调整
- 决策只在边界发生

这就足够了。
